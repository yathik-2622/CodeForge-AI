## Getting Started

### What is CodeForge AI?

CodeForge AI is a **self-hosted autonomous coding agent platform**. Unlike GitHub Copilot or ChatGPT, it:

- **Runs your own AI** — free open-source models (Mistral, Llama, Phi-3) via OpenRouter
- **Connects to your real GitHub repos** — reads actual code, not just descriptions
- **Searches the web** — agents look up docs, CVEs, and best practices in real time
- **Works everywhere** — web browser, VS Code, terminal CLI, and WhatsApp
- **Is 100% self-hostable** — your code never leaves your infrastructure

### Quick Setup Checklist

| Step | What to do | Time |
|------|-----------|------|
| 1 | Deploy CodeForge (Replit, Docker, or VPS) | 5 min |
| 2 | Get OpenRouter API key (free) | 2 min |
| 3 | Create GitHub OAuth App | 3 min |
| 4 | Get Tavily API key (free) | 1 min |
| 5 | Add env vars and restart | 1 min |
| 6 | Sign in with GitHub | 30 sec |

**Total: ~12 minutes to a fully working AI coding agent.**

### Chat — The Core Feature

**Chat** is where you interact with the AI agents. Think of it like a smarter terminal.

#### Starting a Session

1. Click **Chat** in the sidebar
2. Click **+ New Session**
3. Pick a title (e.g., "Fix auth bug") and model
4. Start typing

#### How Streaming Works

Responses stream token by token — you see the AI "think" in real time.  
The agent automatically:
- Detects if your question needs a web search ("latest", "docs", "how to")
- Injects search results into its context
- Formats code in syntax-highlighted blocks

#### Prompt Templates That Work Well

```

Write unit tests for [module name]

Search for best practices for [topic] in [language]

Generate a [REST/GraphQL] API for [entity] with CRUD operations

```

### Repositories

Connect your GitHub repos for context-aware AI responses.

#### Connecting a Repository

**Option A — Via GitHub OAuth (recommended):**
1. Sign in with GitHub (bottom of sidebar)
2. Go to **Repositories → Connect Repository**
3. Search your repos or browse your list
4. Click **Import** next to any repo

**Option B — Manual URL:**
1. Go to **Repositories → Connect Repository**
2. Select **Manual URL** tab
3. Enter the repo name (`owner/repo`) and URL
4. Click **Connect**

#### Repository Scanning

After connecting, click **Scan** to analyze the repo:
- Detects languages, frameworks, databases
- Counts files and lines of code
- Identifies CI/CD configs (GitHub Actions, Docker, Terraform)
- Maps branches

Once scanned, the AI can use this as context in chats.

### Security

The Security page shows all findings from repository scans.

Severity levels:
- 🔴 **Critical** — Hardcoded secrets, SQL injection, RCE vulnerabilities
- 🟠 **High** — Auth issues, exposed tokens, path traversal
- 🟡 **Medium** — Outdated dependencies, weak crypto
- 🔵 **Low** — Code quality issues, missing headers

Click any finding to see:
- Affected file and line number
- Description and exploit scenario
- Recommended fix (with code example)

### Terminal


   ```bash
   npm run package   # creates codeforge-ai-1.0.0.vsix
   ```


## CLI

The CLI lets you use CodeForge AI from any terminal on Windows, Linux, or macOS.

### Installation

**Option 1: npm (global)**
```bash
npm install -g @codeforge/cli

# Verify
codeforge --version
```

**Option 2: From source**
```bash
cd tools/cli
pnpm install
pnpm run build
npm link    # makes 'codeforge' available globally
```

**Windows users:** Use PowerShell or Windows Terminal. The CLI works on Windows natively — no WSL needed.

### Setup

```bash
# Set your server URL (required)
codeforge config --server https://your-codeforge-app.replit.app

# Optional: set default AI model
codeforge config --model meta-llama/llama-3-8b-instruct:free

# Check connection
codeforge status
```

Your config is saved to `~/.codeforge/config.json`.

### Commands

#### `codeforge chat`

Interactive AI chat — your most-used command:

```bash
# Interactive mode
codeforge chat

# One-shot question
codeforge chat "How do I implement JWT refresh tokens in Express?"

# Resume a previous session
codeforge chat --session 5

# Use a specific model for this session
codeforge chat --model google/gemma-3-12b-it:free
```

**In interactive mode:**
- Type your message and press Enter
- The agent streams the response in real time
- Type `exit` or press `Ctrl+C` to quit

#### `codeforge search`

Search the web via Tavily AI:
```bash
codeforge search "React 19 concurrent features"
codeforge search "CVE-2024-1234 severity"
codeforge search "postgres vs mysql performance 2025"
```

#### `codeforge repos`

List connected repositories:
```bash
codeforge repos
```

#### `codeforge scan`

Trigger a repository scan:
```bash
codeforge scan 3    # scan repo with ID 3
```

#### `codeforge sessions`

List recent chat sessions:
```bash
codeforge sessions
```

#### `codeforge models`

Show available free AI models:
```bash
codeforge models
```

### Piping & Scripting

The CLI is pipe-friendly for automation:

```bash
# Ask about a specific file
cat src/auth.ts | codeforge chat "Review this code for security issues"

# Analyze git diff
git diff HEAD~1 | codeforge chat "Explain what changed in this diff"

# Fix a file
cat buggy.py | codeforge chat "Fix all bugs" > fixed.py

# In a bash script
#!/bin/bash
REVIEW=$(codeforge chat "Does this code have any issues: $(cat $1)")
echo "$REVIEW"
```

### Aliases

Add to your `~/.bashrc` or `~/.zshrc`:

```bash
alias cf="codeforge"
alias cfc="codeforge chat"
alias cfs="codeforge search"
alias cfr="codeforge repos"
```

## WhatsApp Integration

Control CodeForge AI from your WhatsApp — send a message, get code back.

### Setup (Twilio)

1. **Create a free Twilio account** at https://twilio.com  
   (Free trial gives $15 credit — enough for thousands of messages)

2. **Enable WhatsApp Sandbox:**
   - Go to Twilio Console → Messaging → Try it out → Send a WhatsApp message
   - Follow instructions to join the sandbox (send a code to the Twilio number)

3. **Set the webhook URL:**
   - In Twilio Console → WhatsApp Sandbox Settings
   - Set **"When a message comes in"** to:
     ```
     https://your-app.replit.app/api/whatsapp/webhook
     ```
   - Method: `HTTP POST`

4. **Add environment variables** to your CodeForge server:
   ```env
   TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   TWILIO_AUTH_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   TWILIO_WHATSAPP_FROM=whatsapp:+14155238886    # Twilio sandbox number
   ```

5. **Restart the server** — check status at `/api/whatsapp/status`

### Using WhatsApp

After setup, send a WhatsApp message to your Twilio sandbox number:

**Basic questions:**
```
How do I implement rate limiting in Express?
What's the difference between JWT and session auth?
Fix this Python error: TypeError: 'NoneType' object is not subscriptable
```

**Code generation:**
```
Write a REST API endpoint for user registration with validation
Generate a React hook for debounced search
Create a Dockerfile for a Node.js app
```

**Web search:**
```
Search for the latest Next.js 15 features
Find the official docs for Tailwind CSS v4
What are the best practices for SQL injection prevention?
```

**Commands:**
| Command | Action |
|---------|--------|
| `/start` or `hi` | Welcome message and feature list |
| `/help` | Show available commands |
| `/new` | Start a fresh conversation |
| `/history` | See your last few messages |

### WhatsApp Tips

- **Context persists** — the AI remembers your conversation within a session (7-day window)
- **Long responses** — the AI splits responses into multiple messages if needed (1500 char chunks)
- **Code blocks** — code is sent as plain text (WhatsApp doesn't support markdown code blocks)
- **Rate limits** — Twilio free tier allows ~1000 messages/month

## AI Models

CodeForge AI uses OpenRouter to access free open-source models.

### Available Models

| Model | ID | Context | Best For |
|-------|-----|---------|---------|
| **Mistral 7B** | `mistralai/mistral-7b-instruct:free` | 32K | General coding, debugging, explanations |
| **Llama 3 8B** | `meta-llama/llama-3-8b-instruct:free` | 8K | Code generation, function writing |
| **Phi-3 Mini** | `microsoft/phi-3-mini-128k-instruct:free` | 128K | Very long files, whole-repo analysis |
| **Gemma 3 12B** | `google/gemma-3-12b-it:free` | 128K | Code analysis, architecture review |

### Choosing the Right Model

- **Default (Mistral 7B):** Best all-around. Fast, good at reasoning, great for everyday tasks.
- **Llama 3 8B:** Excellent at following instructions precisely. Good for code generation tasks.
- **Phi-3 Mini:** Choose this when you need to send very long context (whole files, large repos).
- **Gemma 3 12B:** Best for architectural analysis and understanding complex codebases.

### Adding Premium Models (Optional)

If you add an OpenRouter API key with credits, you can access:
- **Claude 3.5 Sonnet** — Anthropic's best coding model
- **GPT-4o** — OpenAI's multimodal model
- **DeepSeek Coder** — Specialized for code
- **Qwen 2.5 Coder** — Strong on Asian-language code comments

To use: add the model ID in session settings. OpenRouter passes through billing transparently.

## GitHub Integration

### Sign In

Click **Sign in with GitHub** in the sidebar. This requests:
- `repo` — read/write access to your repositories
- `read:user` — your profile info (name, avatar, email)

Your GitHub token is stored securely in the database (encrypt with `pgcrypto` in production).

### Connecting Repositories

After signing in, go to **Repositories → Connect Repository**.  
You'll see a search box that queries your real GitHub repos in real time.

The **Import** button:
1. Creates a record in the local database
2. Scans the repo in the background (detects languages, frameworks, etc.)
3. Makes it available as context for the AI

### Repository Scanning

A deep scan reads:
- `package.json` — NPM dependencies, frameworks, scripts
- `requirements.txt` / `pyproject.toml` — Python packages
- `Dockerfile`, `.github/workflows/` — DevOps tooling
- Full file tree (via GitHub API) — counts files, detects structure

### Using Repos in Chat

When you have a repo connected, the AI can reference it:
```
"Based on this repo, what's the data flow for authentication?"
"What framework does this project use and what are its dependencies?"
```

## Security Scanner

### How It Works

The security scanner analyzes connected repositories for:

1. **Hardcoded secrets** — API keys, passwords, tokens in source code
2. **Injection vulnerabilities** — SQL injection, XSS, command injection
3. **Authentication issues** — weak JWT secrets, missing auth checks
4. **Dependency CVEs** — known vulnerabilities in npm/pip packages
5. **Insecure configs** — exposed endpoints, missing HTTPS, weak CORS

### Triaging Findings

Each finding has a status:
- **Open** — not yet addressed
- **In Progress** — someone is working on it
- **Resolved** — fixed and verified
- **False Positive** — reviewed and not a real issue

Click any finding and ask the AI: *"How do I fix this?"* — it will generate a specific code fix.

## Advanced Tips & Tricks

### Prompt Engineering for Best Results

**Be specific about the output format:**
```
"Generate a TypeScript interface for a User object with fields: id (number), 
email (string), roles (array of 'admin'|'user'|'viewer'), createdAt (Date)"
```

**Give it errors to fix:**
```
"This code throws: TypeError: Cannot read properties of undefined (reading 'map')
Here's the component: [paste code]
Fix the error and explain why it happened."
```

**Request step-by-step reasoning:**
```
"Think step by step: How would you architect a real-time notification system 
for 100,000 concurrent users using Node.js and WebSockets?"
```

**Ask for alternatives:**
```
"Show me 3 different ways to implement pagination in a REST API, 
with pros and cons of each."
```

### Power User Keyboard Shortcuts (VS Code)

| Shortcut | Action |
|----------|--------|
| `Ctrl+Shift+A` | Open CodeForge sidebar |
| `Ctrl+Shift+E` | Explain selected code |
| `Ctrl+Shift+F` | Fix selected code |
| `Ctrl+A` then `Ctrl+Shift+E` | Explain entire file |

### CLI Power Patterns

```bash
# Analyze a PR diff
git diff main...feature-branch | codeforge chat "Review this PR and suggest improvements"

# Check for security issues in a file
codeforge chat "Security review: $(cat sensitive.ts)"

# Generate commit message
git diff --staged | codeforge chat "Write a conventional commit message for these changes"

# Auto-fix a file
codeforge chat "Fix all TypeScript errors: $(cat src/broken.ts)" > src/fixed.ts

# Batch process multiple files
for f in src/**/*.ts; do
  echo "Reviewing $f..."
  codeforge chat "Any issues in this file? $(cat $f)"
done
```

### Session Management Strategy

- **One session per feature** — keeps context focused
- **Name sessions descriptively** — "JWT Auth Bug", "Payment API Refactor"
- **Resume sessions** — the AI remembers the full conversation history
- **Long sessions** — the AI uses the last 10-20 messages as context (older messages are dropped)

### Using Multiple Models

Switch models mid-project for different tasks:
- Start architecture planning with **Gemma 3 12B** (best reasoning)
- Generate boilerplate with **Llama 3 8B** (fast and precise)  
- Debug errors with **Mistral 7B** (great at tracing logic)
- Analyze large files with **Phi-3 Mini** (128K context)

## Troubleshooting

### "Cannot connect to server"

**VS Code / CLI:**
1. Check your server URL is correct: `codeforge status`
2. Make sure the server is running (Replit workflow should show "running")
3. Try opening the URL in your browser: `https://your-app.replit.app/api/health`

**Web app:**
- Open browser DevTools (F12) → Network tab → look for failed requests
- Check for CORS errors (usually means the API server isn't running)

### "AI is not responding"

1. Check OPENROUTER_API_KEY is set in environment variables
2. Verify the key at https://openrouter.ai/keys
3. Free models may be rate-limited — try a different model
4. Check the API server logs for errors

### "GitHub login doesn't work"

1. Verify GITHUB_CLIENT_ID and GITHUB_CLIENT_SECRET are set
2. Check the callback URL in your GitHub OAuth App matches your APP_URL:
   ```
   https://your-app.replit.app/api/auth/github/callback
   ```
3. Make sure APP_URL doesn't have a trailing slash

### "WhatsApp messages aren't getting through"

1. Check `/api/whatsapp/status` — it shows configuration status
2. Verify the Twilio webhook URL is exactly:
   ```
   https://your-app.replit.app/api/whatsapp/webhook
   ```
3. Make sure all three Twilio env vars are set
4. In Twilio console, check the "Message Logs" for errors
5. The sandbox requires users to opt in — send the join code first

### "Web search isn't working"

1. Check TAVILY_API_KEY is set
2. Test at https://app.tavily.com — verify the key is active
3. If Tavily fails, set SERPAPI_KEY as a fallback
4. Without any key, the AI still answers but without live web data

### Database Issues

```bash
# Reset the database schema
pnpm --filter @workspace/db run push-force

# Check database connection
curl https://your-app.replit.app/api/health
```

### Performance

- **Slow responses:** Switch to a faster model (Mistral 7B is fastest)
- **Timeout errors:** Your server may be sleeping (Replit free tier) — open the web app to wake it
- **Large files:** Use Phi-3 Mini for files over 10K lines

## Self-Hosting Guide

### Replit (Easiest)

1. Fork the project on Replit
2. Set secrets in the Secrets tab
3. Click Deploy → Reserved VM

### Docker

```bash
# Clone the repo
git clone https://github.com/your/codeforge-ai
cd codeforge-ai

# Copy and edit env
cp .env.example .env
nano .env

# Start with docker-compose
docker-compose up -d

# Push DB schema
docker-compose exec api pnpm --filter @workspace/db run push-force
```

### VPS (Ubuntu/Debian)

```bash
# Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Install pnpm
npm install -g pnpm

# Clone and install
git clone https://github.com/your/codeforge-ai
cd codeforge-ai && pnpm install

# Setup PostgreSQL
sudo apt install postgresql
sudo -u postgres createdb codeforge

# Configure .env and push schema
pnpm --filter @workspace/db run push-force

# Run with PM2
npm install -g pm2
pm2 start "pnpm --filter @workspace/api-server run start" --name codeforge-api
pm2 start "pnpm --filter @workspace/codeforge run preview" --name codeforge-web
pm2 save && pm2 startup
```

### Windows

```powershell
# Install Node.js 20 from https://nodejs.org
# Install pnpm
npm install -g pnpm

# Clone and install
git clone https://github.com/your/codeforge-ai
cd codeforge-ai
pnpm install

# Set up PostgreSQL (download from https://www.postgresql.org)
# Create database: codeforge

# Copy and edit .env
copy .env.example .env
notepad .env

# Push schema
pnpm --filter @workspace/db run push-force

# Start servers (two terminals)
# Terminal 1:
pnpm --filter @workspace/api-server run dev
# Terminal 2:
pnpm --filter @workspace/codeforge run dev
```
